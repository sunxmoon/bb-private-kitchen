# PRD: bb-private-kitchen UI/UX 优化方案

## 1. 项目背景
“宝宝的私房菜馆”目前功能已基本完善，但在用户体验（UX）和界面布局（UI）上仍有提升空间。部分功能组件位置不当，视觉元素不够协调，管理功能分散。

## 2. 优化目标
- **逻辑重组**：将功能归类到更符合用户直觉的页面。
- **视觉统一**：协调图标、文字、箭头等基础 UI 元素的布局和样式。
- **页面美化**：增强点单页的视觉层次感。
- **集中管理**：构建统一的管理后台，整合用户管理、订单记录和操作日志。
- **质量保障**：引入“审阅测试”角色，确保代码修改后测试覆盖到位。

## 3. 具体需求变更

### 3.1 随机点餐功能迁移
- **变更内容**：将“随机点餐”按钮及弹窗从“菜单页” (`index.html`) 迁移至“点单页” (`order.html`)。
- **理由**：随机选菜是点单的一种前置行为，放在点单流程中更合理。

### 3.2 搜索框布局优化
- **变更内容**：优化 `index.html` 中的搜索框。
    - 调整搜索图标 (`fa-search`) 与文字的间距。
    - 确保输入框在不同尺寸屏幕下的内边距（Padding）协调。
    - 增强焦点（Focus）状态的视觉反馈。

### 3.3 箭头样式与布局标准化
- **变更内容**：统一全站的箭头图标样式。
    - 检查菜单卡片、分页按钮、下拉选择框中的箭头。
    - 统一使用 Font Awesome 且确保垂直居中和间距一致。

### 3.4 点单页样式增强
- **变更内容**：美化 `order.html`。
    - 为输入项增加图标指引。
    - 采用分段式布局（例如：选择菜品、口味偏好、配送详情）。
    - 优化“提交点餐”按钮的视觉冲击力。

### 3.5 管理页面整合
- **变更内容**：
    - 将原来的 `users.html`（用户管理）扩展为“管理中心”。
    - 将 `history.html` 中的“点单记录”和“操作动态”迁移至管理中心，以标签页（Tabs）形式呈现。
    - `history.html` 可保留作为普通用户的“美食统计”页面，或根据需要精简。
    - 更新导航栏，增加明显的“管理”入口（仅限管理员或对所有家庭成员可见，视当前逻辑而定）。

### 3.6 审阅测试角色
- **职责描述**：在每个功能模块开发完成后，必须有专门的测试审查步骤。
- **执行要求**：
    - 对应功能变更必须更新或新增测试用例（`tests/` 目录下）。
    - 验证迁移后的路由是否正确跳转，CSRF Token 是否失效。

## 4. 技术规范
- **框架**：继续使用 Tailwind CSS。
- **图标**：统一使用 Font Awesome 6.x。
- **交互**：保持 HTMX 的无刷新体验。

## 5. 实施计划
1.  **第一阶段**：整合管理中心，迁移用户、记录、日志功能。
2.  **第二阶段**：优化点单页，迁移随机点餐功能。
3.  **第三阶段**：全局 UI 细节微调（搜索框、箭头）。
4.  **第四阶段**：测试代码审查与补充。
